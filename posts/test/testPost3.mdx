## 5. Firebase Data Connect(환호)(비공개 미리보기)

- Google Cloud SQL에 추가되어 있는 postgreSQL을 Firebase를 통해 앱과 연결하는 기술

```
type User @table {
  first: String!
  last: String!
  email: String!
  country: String!
}

query GetUsers @auth(level: USER) {
  users {
    id
    firstName
    lastName
  }
}
```

위와 같은 정보에 기반해 Data Connect는 아래 내용을 생성함

- Cloud SQL의 PostgreSQL DB
- Google 인프라에서 실행되는 API 서버
- 모바일 및 웹앱에서 사용할 수 있는 SDK

DB, API 서버, SDK가 모두 데이터 모델, 쿼리 등에서 생성 ⇒ 쿼리 정의 인프라 ⇒ 항상 서로 동기화됨

스키마 변경 → Data Connect가 DB를 마이그레이션 함 → API 서버 업그레이드 → 새 SDK 생성 ⇒ 전체 자동 진행

### 4-1. 특징

- Type-Safe libraries(정적 타입형 라이브러리)

  ```kotlin
  import davideast.DemoConnector
  import davideast.ListUsersQuery

  val connector = DemoConnector.instance
  val response = connector.listUsers.execute()
  val firstUser = response.data.users.first()
  val firstName = frstUser.first
  ```

  ⇒ 응답값에 String이 없음
  ⇒ 모든 응답값은 Data Connect에 의해 생성된 정적 타입형 코틀린 코드임

- Model Changes
  ```kotlin
  type User @table {
    ~~first: String!~~
    ~~last: String!~~
    firstName: String!
    lastName: String!
    email: String!
    country: String!
  }
  ```
  ⇒ 만약 위와 같이 컬럼 명이 바뀌는 경우
  ⇒ Data Connect가 Postgres에서 DB 마이그레이션을 자동으로 실행
  ⇒ 백엔드에서 프론트로 자동 동기화 → API 서버 업데이트 → 새로운 SDK 생성
- Adding related data

  ```kotlin
  type User @table {
    first: String!
    last: String!
    email: String!
    country: String!
  }

  type Order @table {
  	customer: User!
  	total: Float!
  	name: String!
  }
  ```

  ⇒ 위와 같이 Join 연결이 필요한 경우 자동으로 관계(n:1, n:m)가 연결된다.

## 6. Firebase Genkit

프롬프트를 작성하고 반복하는데 익숙 ⇒ 발전된 생성형 AI 워크플로우가 있다면?

- 세련된 생성형 AI 플로우를 쉽고 간편하고, 익숙한 방법으로 생성하는데 도와주는 오픈소스 프레임워크
- Genkit을 사용해 다음과 같은 기능을 제공하는 AI 기반 앱을 개발할 수 있다.
  - 내가 가진 데이터를 기반으로 질문에 답을 해주는 기능
  - 결정을 내려서 액션을 수행
  - 비정형 데이터를 통찰력으로 전환
  - 우리가 개발한 사용자 경험성을 모든 개별 사용자에게 맞춤화
- 프로토타입 단계에서 운영 단계에 이르기까지 모든 과정 지원
- AI 라이브러리가 이미 일부 Firebase 서비스에 연결됨.
  ⇒ 생성형 AI 워크플로우를 디버깅하는데 도움이 되는 AI 모델, 벡터 저장소(vector stores), 평가 도구 및 기능
  ⇒ Cloud Functions, Cloud Run 등으로 확장 가능
  ⇒ 현재 Cloud Monitoring, Cloud Logging, Firebase Authentication, App Check, Firestore 등으로 통합 가능
- 현재 TypeScript 기반에 서버 사이드 개발에 대해서 지원이 이루어지고 있음 ⇒ 조만간 Go도 지원 예정
- 주요 기능
  - 여러 모델, 하나의 인터페이스
  - 구조화된 출력
  - 멀티모달, 멀티미디어
  - LLM에 도구 제공
  - Dotprompt를 사용한 프롬프트 관리
  - 로컬에서 흐름 실행
  - 트레이스 검사
