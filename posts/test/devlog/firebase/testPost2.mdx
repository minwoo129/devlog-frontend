# 공통사용 기능 코드 작성 규칙 도입

## 1. 계기

- 만세력 시작 전 구르미랑 프로젝트 개발 당시 개발환경으로 인한 여러 문제점들이 확인됨
  <aside>

      1. JavaScript만을 사용한 코드 작성 ⇒ 런타임 에러 발생 가능성이 올라감 

      2. 일부 코드에 대한 규칙 부재 ⇒ 개발자 별 일관되지 않은 코드 설계 방향 ⇒ 유지보수 역량 저하에 따른 생산성 저하 

      3. 일부 함수에 대해 너무 많은 기능이 적용됨 ⇒ 함수의 단일책임 원칙 위배 ⇒ 코드 신뢰도 저하 

      4. 검증되지 않은 오픈소스 라이브러리 사용 ⇒ 프로젝트 환경 업데이트 시 해당 환경과 호환이 안될 수 있음 ⇒ 유지보수에 어려움이 따륾

  </aside>
  ⇒ 이러한 문제는 구르미랑 프로젝트에서 인력 교체가 자주 이루어지면서 심각한 문제가
  되었음
- 위 문제점들을 바탕으로 코드를 작성할 때 규칙을 도입하고 모든 개발자가 따름으로서 개발환경을 개선할 수 있다고 생각하였고 이 중 일부에 대해 만세력 프로젝트에 처음 적용하게 됨.

## 2. 규칙 도입을 위한 진행과정

1. 구르미랑 프로젝트 2년간의 Git 커밋 기록 분석
2. 문제점 분석
   - 화면마다 프로세스가 너무 재각각 ⇒ 유지보수 시간 ⬆️
   - 함수 및 컴포넌트의 응집도가 너무 재각각 ⇒ 유지보수 시간 ⬆️
   - 신규 기능 추가 시 코드 리팩토링이 거의 이루어지지 않음
   - JavaScript와 TypeScript가 혼합해서 사용 ⇒ 관련 코드들이 너무 흩뿌려져 관리가 어려움
   - 기능에 관한 부가설명이 거의 작성되지 않음 ⇒ 인수인계를 위한 코드 이해 불가능

## 3. 규격화된 코드 작성을 위한 액션 아이템(일부 예시)

<aside>

1. 메인언어로 TypeScript를 적극 사용할 것
2. JSDOC 주석 작성을 통해 내용을 파악하는 시간을 줄일 것
3. 함수는 단일책임 원칙에 따라 한가지 기능만 수행하도록 설계할 것
4. 공통적으로 사용하는 함수의 경우 Configuration Object 인수 구조를 통해 확장성을 높일 것
5. 상수는 검색이 가능한 이름으로 선언할 것
6. 오픈소스 라이브러리 사용 시 성능적으로 검증된 라이브러리인지 여부를 확인하여 사용할 것

   → 판단기준: npm 패키지 설치현황 데이터, Github Release 현황 등

</aside>
